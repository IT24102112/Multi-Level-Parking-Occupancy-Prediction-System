📡 IoT Simulator

Simulates sensor events based on real KMC parking patterns

← Dashboard
💡 The simulator replays real vehicle entry/exit patterns from 201,212 KMC parking records (Oct–Dec 2025). Every 30 seconds it fires entry/exit events to keep live occupancy aligned with the expected pattern for the current day and hour. This populates parking_logs, updates parking_level, and every 2 hours writes a snapshot to occupancy_log — exactly as real IoT sensors would.

⚙️ How It Works

⏱
Every 30 seconds
Compares current occupancy to the expected pattern for this day and hour. Fires entry or exit events to gradually align with the target.
📋
Writes to parking_logs
Every simulated sensor event writes a row to parking_logs with direction (entry/exit) and timestamp — identical to real IoT output.
📊
Every 10 minutes
A scheduler snapshots current occupancy per level into occupancy_log, which feeds the reports module with fresh data.
🤖
AI model integration
The real-time AI model reads live parking_level data to generate 15-minute predictions, which are stored in the prediction table.
